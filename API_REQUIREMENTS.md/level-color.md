# 4. 大屏统计：朝代核心色彩与文化标签

**功能描述**：
用于首页大屏右侧“核心色彩”区域的动态展示。根据选中的朝代，返回该朝代最具代表性的四种建筑色彩（含色值）以及对应的文化/艺术描述标签，用于渲染色块排版及底部的标签云。

* **请求地址 (URL)**: `/api/dashboard/core-colors`
* **请求方式 (Method)**: `GET`
* **请求参数 (Query)**:
    | 参数名 | 类型 | 必填 | 说明 | 示例 |
    | :--- | :--- | :--- | :--- | :--- |
    | `dynasty` | `String` | 是 | 朝代标识符 | `tang` / `qing` |

### 返回数据示例 (Response JSON)

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "dynastyName": "宋朝",
    "colors": [
      { "name": "青", "hex": "#7fb2c5", "count": 1200 },
      { "name": "绿", "hex": "#5e9c45", "count": 980 },
      { "name": "白", "hex": "#f2eada", "count": 1500 },
      { "name": "褐", "hex": "#704d30", "count": 600 }
    ],
    "cultureTags": ["青綠輝映", "淡雅柔和", "醇和素雅", "建築彩畫"]
  }
}
```

### 字段说明 (Fields)

| **字段名**     | **类型** | **说明**                                       |
| ---------------------- | ---------------- | ------------------------------------------------------ |
| `dynastyName`    | `String`   | 朝代名称，用于逻辑校验                               |
| `colors`         | `Array`    | 核心色彩列表，固定返回 4 个对象（对应前端 2x2 布局） |
| `colors[].name`  | `String`   | 色彩单字名称（如：青、绿、白、褐）                   |
| `colors[].hex`   | `String`   | 色彩的十六进制色值，前端直接绑定到 CSS 样式          |
| `colors[].count` | `Integer`  | 该色彩在该朝代的出现频次（可用于排序或辅助显示）     |
| `cultureTags`    | `Array`    | 文化标签列表，返回 4-6 个反映该朝代色彩审美的短词    |



# 5. 大屏统计：朝代色彩使用分析（雷达图）

**功能描述**：
用于首页大屏右上角“色彩使用分析”雷达图的数据渲染。从色彩偏好（红、绿、青、黄）以及视觉特征（饱和度、明度）六个维度，量化展示当前朝代建筑色彩的整体性格。

* **请求地址 (URL)**: `/api/dashboard/color-analysis`
* **请求方式 (Method)**: `GET`
* **请求参数 (Query)**:| 参数名      | 类型       | 必填 | 说明       | 示例                |
  | :---------- | :--------- | :--- | :--------- | :------------------ |
  | `dynasty` | `String` | 是   | 朝代标识符 | `tang` / `qing` |

### 返回数据示例 (Response JSON)

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "dynastyName": "唐朝",
    "indicators": [
      { "name": "黄色使用", "value": 85, "max": 100 },
      { "name": "饱和度", "value": 70, "max": 100 },
      { "name": "明度", "value": 75, "max": 100 },
      { "name": "青色", "value": 60, "max": 100 },
      { "name": "绿色", "value": 65, "max": 100 },
      { "name": "红色", "value": 90, "max": 100 }
    ]
  }
}
```

### 字段说明 (Fields)

| 字段名                 | 类型        | 说明                                                     |
| ---------------------- | ----------- | -------------------------------------------------------- |
| `dynastyName`        | `String`  | 当前分析的朝代名称                                       |
| `indicators`         | `Array`   | 雷达图指标数组，固定返回 6 个维度                        |
| `indicators[].name`  | `String`  | 维度名称（对应雷达图的六个顶点）                         |
| `indicators[].value` | `Integer` | 维度的得分/权重值（0-100）                               |
| `indicators[].max`   | `Integer` | 该维度的最大上限值（固定为 100，便于前端统一坐标轴刻度） |

```
---

### 💡 前端对齐小提示：
1. **维度顺序**：雷达图对维度的顺序很敏感，请告诉后端按照“顺时针”或者固定的顺序返回这 6 个 `name`，否则切换朝代时雷达图的形状会发生“错位旋转”。
2. **数据标准化**：后端返回的 `value` 建议统一缩放到 **0-100** 之间，这样你在写 Echarts 配置时就不需要为每个朝代单独去算坐标轴高度了。
```

* **请求方式 (Method)**: `GET`

  | :---------- | :--------- | :--- | :--------- | :------------------ |
  | `dynasty` | `String` | 是   | 朝代标识符 | `tang` / `qing` |

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "dynastyName": "清朝",
    "levels": [
```

# 7. 大屏统计：多朝代建筑等级分布对比 (堆叠柱状图)

**功能描述**：
用于首页大屏底部“各朝代建筑等级分布对比”图表的渲染。该接口返回唐、宋、元、明、清五个朝代的纵向对比数据，每个朝代包含五个等级（皇家、王公、官员、富户、平民）的建筑数量，前端使用堆叠柱状图（Stacked Bar Chart）进行展示。

* **请求地址 (URL)**: `/api/dashboard/dynasty-comparison`
* **请求方式 (Method)**: `GET`
* **请求参数 (Query)**: 无（固定返回主流五个朝代的全量对比数据）

```json

  "data": {
    "timeline": ["唐", "宋", "元", "明", "清"],
    "series": [
      {
        "rankName": "皇家",
        "data": [900, 950, 901, 1100, 1400],
        "color": "#f3b746"
      },
      {
        "rankName": "王公",
        "data": [200, 220, 201, 250, 280],
        "color": "#5e9c45"
      },
      {
        "rankName": "平民",
        "data": [180, 190, 191, 210, 230],
        "color": "#333333"
      },
      {
        "rankName": "富户",
        "data": [110, 120, 101, 130, 150],
        "color": "#666666"
      },
      {
        "rankName": "官员",
        "data": [300, 320, 301, 350, 400],
        "color": "#3c8dbc"
      }
    ],
    "description": "展示唐、宋、元、明、清五个朝代的建筑等级分布情况，数据基于历史文献记载和现存建筑统计。"
  }
}
```

### 字段说明 (Fields)

| 字段名                | 类型       | 说明                                                                       |
| --------------------- | ---------- | -------------------------------------------------------------------------- |
| `timeline`          | `Array`  | X 轴类目数据，即对比的朝代列表                                             |
| `series`            | `Array`  | 堆叠数据系列，每个对象代表一个社会等级                                     |
| `series[].rankName` | `String` | 等级名称（对应图例 Legend）                                                |
| `series[].data`     | `Array`  | 数值数组，长度必须与 `timeline` 一致（依次对应唐、宋、元、明、清的数值） |
| `series[].color`    | `String` | 建议后端返回该等级对应的固定 UI 色值，确保大屏视觉统一                     |
| `description`       | `String` | 底部的数据说明文本                                                         |

---

### 💡 给后端队友的逻辑指引：

这个接口的 SQL 逻辑大概是：
`SELECT dynasty, level, COUNT(*) FROM buildings GROUP BY dynasty, level`
然后后端需要把查询到的结果，按照朝代顺序拆解并重新组装成上面这种按“等级”分类的 `series` 结构。

```json
    "dimensions": ["工艺难度", "成本指数", "稀有度"],
    "materials": [
      {
        "name": "黄色琉璃瓦",
        "values": [95, 98, 92],
```
